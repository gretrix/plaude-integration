REST API endpoints for Plaud Integration

Dashboard Diet Tracking Tasks Contacts API Docs

Overview

The Plaud Integration API provides endpoints for managing diet tracking, tasks, and CRM contacts. All endpoints return JSON responses.

Base URL: http://localhost:5000

Web Pages

GET /

Main dashboard with statistics and recent entries

GET /diet

Diet tracking page with filtering options

GET /tasks

Task management page with sorting and filtering

GET /crm

CRM contacts page with search functionality

API Endpoints

Diet Endpoints

GET /api/diet

Retrieve diet entries with optional filtering

Query Parameters: date, limit, offset

GET /api/diet?date=2025-11-20&limit=10 Response: { "status": "success", "count": 5, "data": [...] }
POST /api/diet

Add new diet entry (from Zapier webhook)

POST /api/diet Body: { "food": "Chicken Salad", "food_type": "Meal", "estimated_calories": 450, "time_of_day": "12:30:00", "date": "2025-11-20" }

Task Endpoints

GET /api/tasks

Retrieve tasks with optional filtering

Query Parameters: status, limit, offset

POST /api/tasks

Add new task (from Zapier webhook)

POST /api/tasks Body: { "task_name": "Review budget", "task_type": "Meeting", "status": "Pending", "responsible_party": "John", "best_due_date": "2025-11-25" }
PATCH /api/tasks/:id/status

Update task status (used by checkbox toggle)

PATCH /api/tasks/1/status Body: { "status": "Completed" } Response: { "status": "success", "message": "Task status updated", "task_id": 1, "new_status": "Completed" }

CRM Endpoints

GET /api/crm

Retrieve contacts with optional search

Query Parameters: search, limit, offset

POST /api/crm

Add new contact (from Zapier webhook)

POST /api/crm Body: { "contact_name": "Alice Johnson", "company": "Tech Corp", "email": "alice@techcorp.com", "phone": "555-0101", "status": "Lead" }

System Endpoints

GET /api/stats

Get dashboard statistics

GET /api/stats Response: { "status": "success", "data": { "calories_today": 1850, "pending_tasks": 5, "total_contacts": 12 } }
GET /health

Health check endpoint

GET /health Response: { "status": "healthy", "database": "connected" }

Response Codes

200 OK - Request successful

400 Bad Request - Invalid request data

404 Not Found - Resource not found

500 Internal Server Error - Server error

Zapier Integration

This API is designed to work with Zapier webhooks. Configure your Zap to POST data to the appropriate endpoints when new transcripts are processed by Plaud.

Note: For production deployment, use your EC2 server URL or ngrok tunnel URL instead of localhost.
